# Changelog

## [1.8.0]

### Added
- **Account tags & triggers tools + skill coverage.** Exposed the `account_to_actors` mechanism — the backend's way of attaching actors to accounts — as curated tools. **`saveAccountActors`** (POST `/papi/1.0/accounts/actors/{nameId}/{currencyId}`) links **Tags**-form actors (account tags) or **AccountTriggers**-form actors (balance / transaction-count triggers) to a workspace `(name, currency)` pair or — via `accountId` — to one specific account; the call REPLACES the linked set, so the tool description mandates scoping with `formId` (an unscoped pair-level call wipes the pair's tags AND pair-level triggers alike; only per-account bindings survive). **`getDataFieldActorsByActor` / `saveDataFieldActorsByActor` / `getDataFieldActorsByForm` / `saveDataFieldActorsByForm`** manage DATA triggers — `AccountTriggers` actors with `valueType=data` + `operator`/`comparisonValue` bound to a data field of one actor or of every actor of a form (the two same-named `formId` slots on the form route are disambiguated via `Param.Wire` as `triggerFormId`). Extended **`getAccounts`** with the backend's `tag` (filter by tag-actor UUID), `ungrouped`, `withTags`, and `withTriggers` params. **Backend dependency (pong-server, same change):** declared the five missing `operationId`s on the public `/papi/1.0/accounts/actors/*` routes and re-dumped `papi-openapi.json` so the drift gate passes. **Skill:** new "Account tags & triggers" section in `simulator-finance` (system-form discovery via `getForms(formTypes="system")`, the `AccountTriggers` data-field schema incl. the `[{title,value}]` select shape, zone-transition firing semantics — `lowerZone`/`normalZone`/`upperZone`, fires only on zone change — webhook delivery to users with access to the trigger actor, the replace-semantics warning, and a `dataField`-is-a-field-id note), plus tag/trigger activation phrases in the frontmatter and a matching section in `docs/entities/accounts.md`. Added four eval scenarios + dry fixtures for the new tools. Updated the README / `docs/ARCHITECTURE.md` tool tables (new "Account tags & triggers" row; also added the previously-unlisted `createAccountPair` to the Accounts row) and the curated-tool figure to ~100. Bumped manifests to 1.8.0.
